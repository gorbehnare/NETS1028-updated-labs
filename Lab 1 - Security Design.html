2. Describe how to prevent removable media filesystems from being automatically mounted.

udisks2 is the service for automounting removable media.

# Stop the service: 
sudo service udisks2 stop
# Disable the service: 
sudo systemctl disable udisks2
# Stop the service: 
sudo service udisks2 stop
# Disable the service: 
sudo systemctl disable udisks2

The service can be manually started. However, if service is masked, then the service cannot be manually started:
systemctl mask udisks2
to unmaks and allow staring the service manually or enabling:
systemctl unmask udisks2
Shortest/best answer Sources: https://unix.stackexchange.com/questions/333721/how-to-stop-auto-mounting-of-devices-in-ubuntu

2. Auto-removing old kernels:

Old kernels can be usually removed using sudo apt autoremove
Another utility is sudo purge-old-kernels which has the same effect as running sudo apt autoremove
This still needs to be run manually after each kernel update. For further automation unattended-upgrades can be used and configured to automatically remove old kernel after an unattended upgrade operation:
Source: https://help.ubuntu.com/community/RemoveOldKernels

2. Other than service control commands like you used in the previous step, describe at least one other way to determine what services your server may be providing?

Programs can be executed on the system without needing to have a "service unit" in systemd. One simple method of finding services that a system may be providing is by looking at the processes that are listening on network ports. This can be achieved using ss utility (netstat may still work, but is being deprecated). If the running process does not open a network port then this method is not useful, and maybe running processes must be investigated using ps, top and other such utilities.
The ss utility options that lists all sockets:

ss -a # display all sockets
ss -l # display listening sockets
ss -a # display all sockets
ss -l # display listening sockets

User Access

1. What Linux user accounts are on your server now? For each account you found, describe its purpose.

Two different methods for getting all user accounts:

  1. using sudo getent passwd
  2. Using grep or reading /etc/passwd file directly (this is for local machine Linux users only, not AD, LDAP, SAMBA, etc.)
    These methods lists all accounts, however does not show which accounts can actually sign into the system

2. Which accounts can be used to log in by end users (i.e. they have a user-oriented general purpose shell and are not locked)?

f an account does not have a valid default shell, the account cannot be used to log in to the system
An account can also have a "locked" password. This is achieved by adding "!" or "*" to the password field in the /etc/shadow file. One method of listing locked accounts is using grep as follows:
sudo grep '!' /etc/shadow && sudo grep '*' /etc/shadow
OR:
sudo grep '!\|*' /etc/shadow
OR
sudo grep -e '!' -e '*' /etc/shadow
Additional information about system users and groups can be found here: https://wiki.debian.org/SystemGroups
